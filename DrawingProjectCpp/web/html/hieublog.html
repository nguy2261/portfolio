Code Snipget 2: Implementation of Command Line

For this feature, I create a new method for Mia to invoke whenever the program input exists.

First, we validate the input picture file and then load it into an internal buffer of Mia:

if (isValidImageFileName(argv[1]))   
{

   if (isValidImageFile(argv[1]))

        {

            m_commandBuffer = ImageHandler::loadImage(argv[1]);

        }

        else

            perror ("Incorrect input file ");

}

else

    perror ("Incorrect input filename ");

Then, we check the second input to see if it’s “-h” or not. If it asked to display the help menu, we display the menu by loading an existing text file and exit the operation.

if (strcmp(argv[1], "-h") == 0)

   {

       system("cat src/help.txt");

       return;

   }

If not, we go into the loop that Mia will iterate through each command input and conduct the requested operation. For example, if the user requested to apply a sharpen filter with an amount of 5.0 to the input picture, Mia will create a filter by the filter factory, apply it, increase the counter to the next command input and move into the next argument.

int counter = 2;               

while (counter <= argc - 2)

    {

        if (strcmp(argv[counter],"-sharpen") == 0){

            Filter * f = FilterFactory::getFilter(FilterFactory::FILTER_SHARPEN, 1, atof(argv[counter+1]));

            FilterFactory::applyFilter(f, m_commandBuffer);

            delete f;

            counter++;

        }

The other command inputs are also handled in the same manner. In the end, Mia will validate the output file and save it as the requested name.

        if (ImageHandler::saveImage(argv[argc-1], m_commandBuffer)) {

            std::cout << "Successfully saved image" << std::endl;

        } else {

            std::cout << "Unsuccessfully saved image..." << std::endl;

        }

During the iteration through the command line input, Mia will conduct all the valid requested command up to the invalid one. In that case, it will break the operation, output the error, save the current result under the requested name and exit.

Code Sniget 3: To test all the tools of the FlashPhoto, we first created some expected results as pre-defined picture by creating fake inputs using the applyToBuffer method of each tool. Then, when testing, we will replicate the same tool’s operation on the buffer and compare it to the expected result.

void TestFlash::runTestTool()

{  

    for (int i = 0; i < 7; i++){

        if (i == 5){

            TStamp::setStampBuffer(ImageHandler::loadImage("test/colorCircles.png"));

        }

       

        if (i == 6){

            m_testBuffer = ImageHandler::loadImage("test/snaily.jpg");

        } else {

            m_testBuffer = new PixelBuffer(800,800,ColorData(1,1,0.95));

        }

       

        for (int x = 390; x < 410; x++){

            for (int y = 390; y < 410; y++){

                 m_tools[i]->applyToBuffer(x,800-y,ColorData(220,240,120),m_testBuffer);

            }

        }

        std::string save = "test/current/tool" + to_string(i) + ".jpg";

        ImageHandler::saveImage(save,m_testBuffer);

        m_testBuffer = ImageHandler::loadImage(save);

        std::string load = "test/testtool" + to_string(i) + ".jpg";

        m_expectBuffer = ImageHandler::loadImage(load);

        if(compareBuffer(m_testBuffer,m_expectBuffer)){

           cout << "Testing Tool " << m_tools[i]->getName() << " PASSED.\n";

        } else {

            cout << "Testing Tool " << m_tools[i]->getName() << " FAILED.\n";

         break;

        }

    }

}

Initially, we decided to do some fake inputs and compare the buffer without saving the m_testBuffer. However, I found out that the ImageHandler would slightly modify the color data of the buffer if I saved it, which made the pictures before and after saving were two different pictures. To go around this error, I have to save the m_testBuffer, the load it again and compare it with m_expectBuffer. The same idea is applied to testing each filter.
